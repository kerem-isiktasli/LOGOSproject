# LOGOS 시스템 파이프라인 설계도

## Language Learning Control Engine: 완전한 논리적 흐름 가이드

**문서 목적**: 이 문서는 LOGOS 언어학습 시스템이 어떻게 작동하는지를 일반인도 이해할 수 있도록 설명하면서, 기술적 세부사항도 모두 포함합니다.

**문서 구조**: C4 Model 기반 (Context → Container → Component → Code)

> 참고: 이 문서는 [소프트웨어 아키텍처 문서화 모범 사례](https://www.freecodecamp.org/news/system-architecture-documentation-best-practices-and-tools/)와 [C4 Model 가이드](https://vfunction.com/blog/architecture-diagram-guide/)를 따릅니다.

# Part 1: 개념적 개요 (Conceptual View)

## 1.1 LOGOS란 무엇인가?

LOGOS는 **"당신이 이 언어로 실제로 무엇을 하고 싶으세요?"** 라는 질문에서 시작하는 언어학습 앱입니다.

### 핵심 철학의 전환

```
┌─────────────────────────────────────────────────────────────────────┐
│                    전통적 언어학습 vs LOGOS                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   전통적 접근법                      LOGOS 접근법                     │
│   ─────────────                    ────────────                    │
│                                                                     │
│   "얼마나 많이 아는가?"     →      "어디서 실제로 사용할 수 있는가?"      │
│                                                                     │
│   단어 1,000개 암기          →      CELBAN 시험 통과                  │
│   문법 규칙 학습             →      환자와 의사소통                    │
│   모든 상황에 일반적 학습     →      당신의 목표에 맞춘 학습             │
│                                                                     │
│   측정 기준: 정답률           →      측정 기준: 사용 가능 영역           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 실제 시나리오: Maria의 이야기

**Maria**는 토론토에서 간호사로 일하기 위해 CELBAN 시험을 4개월 안에 통과해야 하는 브라질 출신 간호사입니다.

**첫 번째 실행**: Maria가 LOGOS를 처음 열면:

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│   "LOGOS에 오신 것을 환영합니다. 당신에게 정확히 필요한 것이          │
│    무엇인지 함께 알아보겠습니다."                                     │
│                                                                     │
│   ┌───────────────────────────────────────────────────────────┐     │
│   │                                                           │     │
│   │  [ 구조화된 선택 ]        또는        [ 자연어 입력 ]       │     │
│   │                                                           │     │
│   │  분야: 의료                  "저는 브라질에서 온 간호사이고,│     │
│   │  기술: 읽기 + 듣기           토론토에서 일하기 위해 4개월   │     │
│   │  목적: 자격증                안에 CELBAN을 통과해야 해요."  │     │
│   │  시험: CELBAN                                             │     │
│   │  기한: 4개월                                              │     │
│   │                                                           │     │
│   └───────────────────────────────────────────────────────────┘     │
│                                                                     │
│                    [ 학습 시작하기 ]                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 3.2 핵심 알고리즘 설명

### IRT (Item Response Theory) - 문항반응이론

```
╔═══════════════════════════════════════════════════════════════════════════════╗
║                            IRT 알고리즘 작동 원리                              ║
╠═══════════════════════════════════════════════════════════════════════════════╣
║                                                                               ║
║  목적: "이 학습자가 이 문항을 맞출 확률은?"                                    ║
║                                                                               ║
║  핵심 공식 (2PL 모델):                                                        ║
║  ┌─────────────────────────────────────────────────────────────────────────┐ ║
║  │                           1                                              │ ║
║  │  P(정답) = ─────────────────────────                                     │ ║
║  │            1 + e^(-a(θ - b))                                             │ ║
║  │                                                                          │ ║
║  │  θ (theta) = 학습자 능력 (보통 -3 ~ +3, 0이 평균)                        │ ║
║  │  b = 문항 난이도                                                         │ ║
║  │  a = 변별도 (이 문항이 능력을 얼마나 잘 구분하는가)                        │ ║
║  └─────────────────────────────────────────────────────────────────────────┘ ║
║                                                                               ║
║  예시: Maria (θ = -0.5)가 "contraindication" (b = 1.2)을 접할 때:            ║
║                                                                               ║
║  ┌─────────────────────────────────────────────────────────────────────────┐ ║
║  │                            1                                             │ ║
║  │  P(정답) = ───────────────────────────                                   │ ║
║  │            1 + e^(-1.0 × (-0.5 - 1.2))                                   │ ║
║  │                                                                          │ ║
║  │          = 1 / (1 + e^1.7)                                               │ ║
║  │          = 1 / (1 + 5.47)                                                │ ║
║  │          = 0.18 (18%)                                                    │ ║
║  │                                                                          │ ║
║  │  해석: Maria가 이 단어를 맞출 확률은 18%에 불과                           │ ║
║  │  → 시스템 판단: 너무 어려움, 힌트 제공 필요                               │ ║
║  └─────────────────────────────────────────────────────────────────────────┘ ║
║                                                                               ║
║  θ 업데이트 (Maximum Likelihood Estimation):                                 ║
║  ┌─────────────────────────────────────────────────────────────────────────┐ ║
║  │  Maria가 (힌트 도움으로) 맞췄을 때:                                       │ ║
║  │                                                                          │ ║
║  │  "예상(18%)보다 높은 확률로 맞췄으므로,                                   │ ║
║  │   Maria의 실제 능력은 -0.5보다 높을 수 있다"                              │ ║
║  │                                                                          │ ║
║  │  Newton-Raphson 반복:                                                    │ ║
║  │  θ_new = θ_old - (∂L/∂θ) / (∂²L/∂θ²)                                   │ ║
║  │  θ_new = -0.5 + 0.05 = -0.45                                            │ ║
║  │                                                                          │ ║
║  │  결과: Maria의 추정 능력이 -0.5 → -0.45로 상승                           │ ║
║  └─────────────────────────────────────────────────────────────────────────┘ ║
║                                                                               ║
╚═══════════════════════════════════════════════════════════════════════════════╝
```

### FSRS (Free Spaced Repetition Scheduler) - 간격 반복 스케줄러

```
╔═══════════════════════════════════════════════════════════════════════════════╗
║                            FSRS 알고리즘 작동 원리                             ║
╠═══════════════════════════════════════════════════════════════════════════════╣
║                                                                               ║
║  목적: "이 단어를 언제 다시 복습하면 최적인가?"                                ║
║                                                                               ║
║  핵심 개념:                                                                   ║
║  ┌─────────────────────────────────────────────────────────────────────────┐ ║
║  │  Stability (S): 기억이 유지되는 기간 (일 단위)                           │ ║
║  │  → S = 10이면 90% 기억률로 10일간 유지                                   │ ║
║  │                                                                          │ ║
║  │  Difficulty (D): 단어의 학습 난이도 (1~10)                               │ ║
║  │  → 높을수록 다음 복습까지 간격이 짧아짐                                   │ ║
║  │                                                                          │ ║
║  │  Retrievability (R): 현재 기억할 수 있는 확률                            │ ║
║  │  R = e^(-t/S)  (t = 마지막 복습 후 경과일)                               │ ║
║  └─────────────────────────────────────────────────────────────────────────┘ ║
║                                                                               ║
║  Rating 시스템:                                                               ║
║  ┌─────────────────────────────────────────────────────────────────────────┐ ║
║  │  Rating │ 의미      │ 효과                                              │ ║
║  │  ───────┼──────────┼──────────────────────────────────────────────────│ ║
║  │    1    │ Again    │ Stability 리셋, 내일 다시 복습                    │ ║
║  │    2    │ Hard     │ Stability 약간 증가, 힌트 사용했음                 │ ║
║  │    3    │ Good     │ Stability 적당히 증가, 정상 복습                   │ ║
║  │    4    │ Easy     │ Stability 크게 증가, 더 긴 간격                    │ ║
║  └─────────────────────────────────────────────────────────────────────────┘ ║
║                                                                               ║
║  예시: "contraindication" 첫 학습 후 (Rating 2 = Hard):                      ║
║  ┌─────────────────────────────────────────────────────────────────────────┐ ║
║  │  Before: S = 0 (신규), D = 5 (기본)                                      │ ║
║  │                                                                          │ ║
║  │  공식 적용:                                                              │ ║
║  │  S_new = S_initial × Rating modifier                                    │ ║
║  │  S_new = 1 × 0.6 = 0.6일 (약 14시간)                                     │ ║
║  │                                                                          │ ║
║  │  D_new = D_old - w₆ × (Rating - 3)                                      │ ║
║  │  D_new = 5 - 0.86 × (2 - 3) = 5 + 0.86 = 5.86                           │ ║
║  │  (Rating이 낮으면 → 더 어려운 단어로 기록)                                │ ║
║  │                                                                          │ ║
║  │  Next Review = Now + S_new = 14시간 후                                   │ ║
║  │                                                                          │ ║
║  │  After: S = 0.6, D = 5.86, nextReview = 내일 아침                        │ ║
║  └─────────────────────────────────────────────────────────────────────────┘ ║
║                                                                               ║
║  시간에 따른 기억률 곡선:                                                     ║
║  ┌─────────────────────────────────────────────────────────────────────────┐ ║
║  │  100% │ ●                                                               │ ║
║  │       │  ╲                                                              │ ║
║  │       │   ╲                                                             │ ║
║  │  80%  │    ●  복습 시점                                                 │ ║
║  │       │     ╲  ↓                                                        │ ║
║  │  60%  │      ╲ ●                                                        │ ║
║  │       │       ╲  ╲                                                      │ ║
║  │  40%  │        ╲  ╲                                                     │ ║
║  │       │         ╲  ● 최적 복습 시점 (90% 유지)                          │ ║
║  │  20%  │          ╲                                                      │ ║
║  │       │           ╲                                                     │ ║
║  │   0%  └──────────────────────────────────────────→ 시간                 │ ║
║  └─────────────────────────────────────────────────────────────────────────┘ ║
║                                                                               ║
╚═══════════════════════════════════════════════════════════════════════════════╝
```

### FRE Priority (빈도-관계성-기여도 우선순위)

```
╔═══════════════════════════════════════════════════════════════════════════════╗
║                          FRE Priority 계산 원리                               ║
╠═══════════════════════════════════════════════════════════════════════════════╣
║                                                                               ║
║  목적: "어떤 단어를 먼저 학습해야 가장 효율적인가?"                             ║
║                                                                               ║
║  핵심 공식:                                                                   ║
║  ┌─────────────────────────────────────────────────────────────────────────┐ ║
║  │            (w_F × F) + (w_R × R) + (w_E × E)                             │ ║
║  │  Priority = ─────────────────────────────────                            │ ║
║  │                         Cost                                             │ ║
║  │                                                                          │ ║
║  │  F = Frequency (빈도): 목표 텍스트에서 얼마나 자주 나오는가?              │ ║
║  │  R = Relational Density (관계성): 다른 단어와 얼마나 연결되어 있는가?     │ ║
║  │  E = Contextual Contribution (기여도): 문맥 이해에 얼마나 중요한가?       │ ║
║  │  Cost = 학습 비용: 이 사용자가 이 단어를 배우는 데 드는 노력              │ ║
║  │                                                                          │ ║
║  │  Default weights: w_F = 0.4, w_R = 0.3, w_E = 0.3                        │ ║
║  └─────────────────────────────────────────────────────────────────────────┘ ║
║                                                                               ║
║  Cost 계산:                                                                   ║
║  ┌─────────────────────────────────────────────────────────────────────────┐ ║
║  │  Cost = BaseDifficulty - TransferGain + ExposureNeed                    │ ║
║  │                                                                          │ ║
║  │  BaseDifficulty: 단어 자체의 난이도 (길이, 발음, 불규칙성)                │ ║
║  │  TransferGain: L1(모국어)에서의 전이 이점                                │ ║
║  │    예: 포르투갈어 "contra" → 영어 "contra-" (전이 이득 0.2)              │ ║
║  │  ExposureNeed: 추가 노출이 필요한 정도                                    │ ║
║  └─────────────────────────────────────────────────────────────────────────┘ ║
║                                                                               ║
║  예시 비교:                                                                   ║
║  ┌─────────────────────────────────────────────────────────────────────────┐ ║
║  │  단어             │  F    │  R    │  E    │ Cost │ Priority │ 순위     │ ║
║  │  ─────────────────┼───────┼───────┼───────┼──────┼──────────┼─────────│ ║
║  │  administer       │ 0.89  │ 0.76  │ 0.82  │ 0.30 │   8.2    │   1위   │ ║
║  │  → 이유: 허브 단어, 12개 단어와 연결, 학습 비용 낮음                      │ ║
║  │                   │       │       │       │      │          │         │ ║
║  │  assess           │ 0.91  │ 0.68  │ 0.79  │ 0.35 │   6.8    │   2위   │ ║
║  │  → 이유: CELBAN 89%에 등장, 매우 고빈도                                  │ ║
║  │                   │       │       │       │      │          │         │ ║
║  │  contraindication │ 0.67  │ 0.45  │ 0.91  │ 0.60 │   3.4    │   3위   │ ║
║  │  → 이유: 중요하지만 비용이 높음 (긴 단어, 의료 전문어)                    │ ║
║  └─────────────────────────────────────────────────────────────────────────┘ ║
║                                                                               ║
║  숙달 조정 (Mastery Adjustment):                                             ║
║  ┌─────────────────────────────────────────────────────────────────────────┐ ║
║  │  이미 어느 정도 아는 단어는 우선순위가 낮아짐                              │ ║
║  │                                                                          │ ║
║  │  Stage 0 (Unknown):    조정 = 1.2× (신규 아이템 부스트)                  │ ║
```

# Part 9: 용어 사전

| 용어 | 영문 | 정의 |
|------|------|------|
| **GoalSpec** | Goal Specification | 사용자의 언어학습 목표를 구조화한 데이터 |
| **언어 객체** | Language Object | 학습 가능한 최소 단위 (단어, 패턴, 규칙 등) |
| **사용 공간** | Usage Space | 언어를 효과적으로 사용할 수 있는 좌표 (목적, 맥락, 형식, 분야) |
| **F** | Frequency | 목표 관련 텍스트에서의 출현 빈도 |
| **R** | Relational Density | 다른 언어 지식과의 연결 정도 |
| **E** | Contextual Contribution | 의미 파악에 대한 기여도 |
| **Cost** | Learning Cost | 이 사용자가 이 항목을 배우는 데 드는 노력 |
| **θ (Theta)** | Ability Parameter | 학습자의 능력 수준을 나타내는 통계 변수 |
| **IRT** | Item Response Theory | 적응형 테스트를 위한 심리측정 모델 |
| **숙달 단계** | Mastery Stage | Unknown(0)부터 Automatic(4)까지의 5단계 |
| **간격 반복** | Spaced Repetition | 망각을 방지하기 위한 최적 간격 복습 스케줄링 |
| **유창성 엔진** | Fluency Engine | 고빈도 패턴(고PMI)을 자동화하는 훈련 모드 |
| **다양성 엔진** | Versatility Engine | 저빈도 조합(저PMI)으로 창의적 확장을 훈련하는 모드 |
| **스캐폴딩 갭** | Scaffolding Gap | 힌트 사용 정확도와 무힌트 정확도의 차이 |
| **PMI** | Pointwise Mutual Information | 단어 동시출현 강도를 측정하는 지표 |
| **병목** | Bottleneck | 학습 진행을 막는 특정 언어 영역 |
